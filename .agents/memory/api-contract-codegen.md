---
name: API contract codegen
description: How the generated Zod types in lib/api-zod relate to the OpenAPI spec, for anyone adding/changing API fields.
---

`lib/api-zod/src/generated/**` is generated by orval (`lib/api-spec` package, `pnpm run codegen`, config in `lib/api-spec/orval.config.ts`) from the hand-maintained spec at `lib/api-spec/openapi.yaml`. Files under `generated/` say "Do not edit manually" and mean it.

**Why:** the routes (`artifacts/api-server/src/routes/*.ts`) validate responses through these generated Zod schemas via `.parse()`. Zod strips unknown keys by default (schemas aren't `.strict()`), so adding a new DB column without updating the spec doesn't break existing responses -- it just silently omits the new field from the API until the spec is updated.

**How to apply:** if a new/changed DB field needs to be exposed through the API (dashboard, mobile app, etc.), add it to `openapi.yaml` first, then re-run `pnpm --filter @workspace/api-spec run codegen` to regenerate the Zod types (this also runs `typecheck:libs`). Backend-only fields (not needed by any API consumer) can skip this entirely -- the DB schema and the API contract are allowed to diverge as long as nothing needs the field surfaced.
