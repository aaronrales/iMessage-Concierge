## Authentication

Every API request requires these two headers:
```
sb-api-key-id: YOUR_API_KEY
sb-api-secret-key: YOUR_API_SECRET
Content-Type: application/json
```

**Important**: All requests must come from a backend server — Sendblue blocks browser/frontend requests.

## Send a Message

```bash
curl -X POST https://api.sendblue.co/api/send-message \
  -H "sb-api-key-id: YOUR_API_KEY" \
  -H "sb-api-secret-key: YOUR_API_SECRET" \
  -H "Content-Type: application/json" \
  -d '{
    "number": "+1XXXXXXXXXX",
    "from_number": "YOUR_SENDBLUE_NUMBER",
    "content": "Hello from Sendblue!",
    "media_url": "https://example.com/image.jpg",
    "status_callback": "https://your-server.com/status",
    "send_style": "celebration"
  }'
```

Or via CLI (no keys needed — CLI is already authenticated):

```bash
sendblue send +1XXXXXXXXXX 'Hello from Sendblue!'
```

### Send Message Parameters
- `number` (required): Recipient in E.164 format
- `from_number` (required): Your Sendblue number (from `sendblue lines`)
- `content`: Message text (max 18,996 characters)
- `media_url`: URL of image, video, audio, or file (max 100 MB iMessage, 5 MB SMS)
- `status_callback`: URL to receive delivery status updates
- `send_style`: iMessage effect — `celebration`, `fireworks`, `lasers`, `love`, `confetti`, `balloons`, `invisible`, `gentle`, `loud`, `slam`, etc.

Message statuses: REGISTERED → PENDING → QUEUED → ACCEPTED → SENT → DELIVERED (or DECLINED / ERROR).

## Receive Messages (Webhooks)

Set up a webhook to receive inbound messages:

```bash
sendblue webhooks set-receive https://your-server.com/webhook
```

Sendblue POSTs inbound messages to your webhook URL:

```json
{
  "from_number": "+15551234567",
  "to_number": "YOUR_SENDBLUE_NUMBER",
  "content": "Hey, got your message!",
  "media_url": "https://storage.sendblue.co/...",
  "service": "iMessage",
  "group_id": null,
  "date_sent": "2024-01-15T10:30:00Z"
}
```

Respond with 2xx to acknowledge. Media URLs expire after 30 days.

Webhook types: `receive`, `outbound`, `typing_indicator`, `call_log`, `line_blocked`, `line_assigned`, `contact_created`.

## SDKs

### TypeScript
```bash
npm install sendblue
```
```typescript
import SendblueAPI from 'sendblue';

// Use values from `sendblue show-keys`
const client = new SendblueAPI({
  apiKey: 'YOUR_API_KEY',
  apiSecret: 'YOUR_API_SECRET',
});

// Send a message (from_number from `sendblue lines`)
const res = await client.messages.send({
  content: 'Hello from Sendblue!',
  from_number: 'YOUR_SENDBLUE_NUMBER',
  number: '+1XXXXXXXXXX',
});

// Read receipt
await client.messages.sendReadReceipt({ from_number: 'YOUR_SENDBLUE_NUMBER', number: '+1XXXXXXXXXX' });

// Typing indicator
await client.messages.sendTypingIndicator({ from_number: 'YOUR_SENDBLUE_NUMBER', number: '+1XXXXXXXXXX' });

// Lookup
const lookup = await client.contacts.evaluateService({ number: '+1XXXXXXXXXX' });
```

## iMessage Features Available via API
- Send/receive text, images, video, audio, files, voice notes, contact cards
- Typing indicators ("..." bubble)
- Read receipts
- Tapback reactions (love, like, dislike, laugh, emphasize, question)
- Expressive effects (celebration, fireworks, lasers, invisible ink, etc.)
- Group messaging (create groups, add members)
- Carousel messages (2–20 swipeable images)
- Contact card delivery (vCard via API)
- FaceTime Audio calls via API
- iMessage detection per phone number

### Fallback Behavior
When a recipient doesn't have iMessage: Sendblue tries RCS first, then SMS. This cascade — iMessage → RCS → SMS — ensures 100% reachability at no additional cost.

### Supported Media
All file types supported. Send `.caf` files for inline voice notes. Send `.vcf` files for contact cards. Convert audio: `ffmpeg -i input.mp3 -acodec opus -b:a 24k output.caf`.

## Rate Limits

- 1 message/second per dedicated number
- Messages queue automatically (10 messages at once = 10 seconds to process)
- AI Agent plan: 1,000 inbound contacts/day/line, 200 follow-ups/day/line, unlimited replies within 24h
- Blue Ocean plan: 50 new outbound contacts/day/line, 200 follow-ups/day/line, 5 messages max without a reply
- Queue holds 1,500 messages (429 after)
- iMessage detection: 30/hour, 100/day per line
- Contacts API: 100 requests/10 seconds
- Need higher limits? Add more phone lines in Settings → Phone Lines to proportionally increase capacity.
